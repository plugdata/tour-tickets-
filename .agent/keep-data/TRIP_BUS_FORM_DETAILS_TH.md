# รายละเอียด TRIP & BUS ROUND FORM - ภาษาไทย

## 📍 ส่วนที่ 1: TRIP MANAGEMENT (การจัดการทริป)

### หน้า: `trips/list.html`

---

### 📋 ฟอร์มเพิ่ม/แก้ไขทริป (Add/Edit Trip Form)

#### ฟิลด์ที่ 1: **ชื่อทริป** (Title)
```
ชื่อ: Title
ประเภท: Text (ข้อความ)
ความยาว: 100-150 ตัวอักษร
บังคับกรอก: ใช่ (Required)
```
**ทำหน้าที่**: ชื่อของทัวร์/แพ็คเกจ ที่แสดงบนเว็บไซต์
**ตัวอย่าง**: 
- "ภูเก็ต 3 วัน 2 คืน"
- "เกาะลันตา ทัวร์สัตว์ป่า"
- "เชียงใหม่ ผู้ชี้ชาว (3D2N)"

**ข้อควรระวัง**: 
- ❌ ห้ามซ้ำชื่อกับทริปเดิม
- ✓ ควรชัดเจน สั้น ไม่ยาวเกินไป
- ✓ ใช้ชื่อภาษาไทยที่ลูกค้าเข้าใจง่าย

---

#### ฟิลด์ที่ 2: **รายละเอียด** (Description)
```
ชื่อ: Description
ประเภท: Text Area (ข้อความยาว)
ความยาว: ไม่จำกัด (ยิ่งละเอียดยิ่งดี)
บังคับกรอก: ไม่บังคับ (Optional)
```
**ทำหน้าที่**: อธิบายรายละเอียดของทริป ให้ลูกค้าเข้าใจ
**ควรระบุ**:
- 📍 สถานที่ที่จะไป
- ⏱️ เวลาออกเดินทาง
- 🚗 ความยาวทริป (กี่วัน)
- 🏨 ที่พัก (ระดับ/ชื่อ)
- 🍽️ อาหารที่รวมอยู่
- 👥 ความเหมาะสม (ครอบครัว/คู่รัก/ผู้ใหญ่)
- ⚠️ ข้อสังเกต (จะกระเบิด น้ำ ห้อม)

**ตัวอย่าง**:
```
ทริปภูเก็ตยอดนิยม! ออกเดินทาง วันศุกร์-อาทิตย์
- วันที่ 1: เก็บตัวที่ชายหาดแก่ง (บุฟเฟ่ต์อาหารทะเล)
- วันที่ 2: ชมหมู่เกาะ + สัตว์ป่า
- วันที่ 3: ซ้อมน้ำ + พักผ่อน
รวมที่พัก 3 ดาว + อาหาร 6 มื้อ
เหมาะสำหรับ: ครอบครัว ผู้ใหญ่ โปรแกรมไม่ยากเกินไป
ข้อสังเกต: มีจุดน้ำจืด 1 โลเซ็นต์ หากแพ้ควรหลีกเลี่ยง
```

---

#### ฟิลด์ที่ 3: **ราคา** (Price)
```
ชื่อ: Price
ประเภท: Number (ตัวเลข)
หน่วย: บาท
บังคับกรอก: ใช่ (Required)
ช่วงที่ยอมรับ: > 0
```
**ทำหน้าที่**: ราคาปกติของทริป (ต่อคน)
**ความสำคัญ**: 
- 🔴 ตัวเลขนี้เป็นราคาฐาน
- บวกกับ "ราคาเพิ่มเติม" จาก Bus Round ได้

**ตัวอย่าง**:
- Trip ราคา: 3,500 บาท/คน
- Bus Round ราคาเพิ่ม: 500 บาท (ถ้ารีสอร์ตแพง)
- **ลูกค้าจ่าย**: 3,500 + 500 = 4,000 บาท

**ข้อควรระวัง**:
- ✓ ตั้งราคาให้สมเหตุสมผล (บ้วนแบบ)
- ✓ เช็คราคาจริงจาก supplier ก่อนตั้ง
- ❌ ห้ามปรับราคาหลังจากมีคนจองแล้ว (ใช้ Bus Round ราคาเพิ่มแทน)

---

#### ฟิลด์ที่ 4: **มัดจำ** (Deposit)
```
ชื่อ: Deposit
ประเภท: Number (ตัวเลข)
หน่วย: บาท
บังคับกรอก: ไม่บังคับ (Optional)
ค่าตั้งต้น: 0
```
**ทำหน้าที่**: จำนวนเงินที่ต้องจ่ายล่วงหน้า เพื่อจองทริป
**ความสำคัญ**: 
- 💰 ลดความเสี่ยง (ลูกค้ายกเลิกฉันนี้)
- 📊 ช่วยวางแผนจำนวนคน

**ตัวอย่าง**:
- Trip ราคา: 3,500 บาท
- Deposit: 1,000 บาท (ประมาณ 30%)
- ลูกค้าจ่ายมัดจำ: 1,000 บาท (โอนเสร็จแล้วถือว่าจองแล้ว)
- ลูกค้าจ่ายส่วนที่เหลือ: 2,500 บาท (ก่อนเดินทาง 3-5 วัน)

**ข้อควรระวัง**:
- ✓ ตั้ง Deposit 20-50% ของราคาเต็ม
- ✓ ระบุชัดว่า "หากยกเลิกจะคืน/ไม่คืน มัดจำ"
- ✓ เก็บเงินมัดจำ ASAP (อย่ารอจ่ายเต็ม)

---

#### ฟิลด์ที่ 5: **รูปภาพ** (Image URL)
```
ชื่อ: Image URL / imageUrl
ประเภท: File Upload / Image
บังคับกรอก: แนะนำให้มี
```
**ทำหน้าที่**: รูปแบบนอร์มของทริป ที่แสดงหน้าเว็บไซต์
**ข้อกำหนด**:
- 📐 ขนาดภาพ: 800x600 px ขึ้นไป
- 🎨 รูปที่สวย/น่าสนใจ
- 📱 ต้องดูสวยทั้งมือถือและ desktop

**ตัวอย่าง**:
- รูป: ชายหาดสวย ท้องฟ้าสดใส คน ยิ้ม
- ❌ ห้ามเป็นรูปทั่วไป/เก่า/มืดเมือย
- ✓ ควรคุณภาพสูง ชัดเจน

---

#### ฟิลด์ที่ 6: **ประเภททริป** (Trip Type)
```
ชื่อ: Trip Type
ประเภท: Dropdown (เลือกจากรายการ)
ตัวเลือก:
  - DOMESTIC (ทริปในประเทศ)
  - INTERNATIONAL (ทริปต่างประเทศ)
บังคับกรอก: ใช่ (Required)
ค่าตั้งต้น: DOMESTIC
```
**ทำหน้าที่**: บอกว่าทริปนี้อยู่ไหน เพื่อกรองการค้นหา
**ตัวอย่าง**:
- DOMESTIC: ภูเก็ต เชียงใหม่ เกาะสมุย กระบี่
- INTERNATIONAL: ลาว ม้ยนมาร์ กัมพูชา เวียดนาม

**ข้อควรระวัง**:
- ✓ เลือกให้ถูกตามสถานที่จริง
- ✓ ทริปต่างประเทศสำคัญเพราะเรื่องวีซ่า/เอกสาร

---

#### ฟิลด์ที่ 7: **ประเทศ** (Country)
```
ชื่อ: Country
ประเภท: Text
ตัวอย่าง: "ไทย", "ลาว", "เมียนมา", "กัมพูชา"
บังคับกรอก: ไม่บังคับ (Optional)
```
**ทำหน้าที่**: ระบุประเทศที่ทริปจะไป (ใช้กรองค้นหา)
**ตัวอย่าง**:
- ทริป "เชียงใหม่" → Country: "ไทย"
- ทริป "ลาวใหญ่" → Country: "ลาว"

---

#### ฟิลด์ที่ 8: **ทริปฮิต** (Is Hot)
```
ชื่อ: Is Hot
ประเภท: Checkbox (ติ๊ก/ไม่ติ๊ก)
ตัวเลือก:
  ☑️ Yes (เป็นทริปฮิต)
  ☐ No (ไม่เป็นทริปฮิต)
บังคับกรอก: ไม่บังคับ
ค่าตั้งต้น: No
```
**ทำหน้าที่**: ทริปที่ฮิต จะแสดงบนหน้าแรกเว็บ และจัดเรียงไปด้านหน้า
**เหตุผล**:
- 📈 ทริปที่ขายดี
- 👥 ลูกค้าสนใจ
- 🎯 ต้องการ promote

**ตัวอย่าง**:
- เลือก "Yes" → ทริปนี้ปรากฏ "🔥 HOT" บนเว็บ
- ลูกค้ามองเห็นก่อน ทำให้เลือกก่อน

---

#### ฟิลด์ที่ 9: **ลำดับฮิต** (Hot Order)
```
ชื่อ: Hot Order
ประเภท: Number (ตัวเลข)
ช่วงที่ยอมรับ: 1-99
บังคับกรอก: ไม่บังคับ
ค่าตั้งต้น: 99
```
**ทำหน้าที่**: กำหนดตำแหน่งของทริปฮิต บนหน้าเว็บ (อันไหนแสดงก่อน)
**ตรรมชาติ**: ตัวเลขน้อย = แสดงไปด้านหน้า
**ตัวอย่าง**:
- ทริป A: Hot Order = 1 → แสดงตำแหน่ง 1 (ชิดบนสุด)
- ทริป B: Hot Order = 2 → แสดงตำแหน่ง 2 (ใต้ทริป A)
- ทริป C: Hot Order = 99 → แสดงตำแหน่ง 99 (ชิดล่างสุด)

**ข้อควรระวัง**:
- ✓ ตัวเลขน้อยก่อน = ลูกค้าเห็นก่อน = ขายได้เร็ว
- ✓ โดยปกติตั้ง 1-10 สำหรับทริปฮิต

---

#### ฟิลด์ที่ 10: **เอกสาร/ไฟล์** (Document URL)
```
ชื่อ: Document URL / docUrl
ประเภท: File Upload
ตัวอย่าง: PDF, Word
บังคับกรอก: ไม่บังคับ (Optional)
```
**ทำหน้าที่**: ไฟล์ PDF เพิ่มเติม เช่น แผนที่ รายละเอียด ข้อมูลที่พัก
**ตัวอย่าง**:
- แผนที่เส้นทาง.pdf
- รายชื่อโรงแรม.pdf
- ข้อมูลการประกันภัย.pdf
- ข้อต้องห้าม.pdf

---

#### ฟิลด์ที่ 11: **สถานะ** (Is Active)
```
ชื่อ: Is Active
ประเภท: Checkbox (ติ๊ก/ไม่ติ๊ก)
ตัวเลือก:
  ☑️ Active (ทริปใช้งาน/ยังมีสำหรับจอง)
  ☐ Inactive (ปิดทริป/ลบออก)
บังคับกรอก: ไม่บังคับ
ค่าตั้งต้น: Active
```
**ทำหน้าที่**: 
- ✅ Active = ลูกค้าเห็นและจองได้บนเว็บไซต์
- ❌ Inactive = ซ่อนทริป ลูกค้าจองไม่ได้

**เหตุผลการปิด**:
- ❌ ทริปโปรแกรมจบไปแล้ว
- ❌ โรงแรมไม่มีห้องว่าง
- ❌ Supplier ปรับโปรแกรม
- ❌ ทัวร์ยกเลิก

**ข้อควรระวัง**:
- ✓ ห้ามลบทริปทันที (เก็บไว้ record ประวัติ)
- ✓ ใช้ "Inactive" แทนการลบ

## 📍 ส่วนที่ 2: BUS ROUND MANAGEMENT (การจัดการรอบรถ)

### หน้า: `bus-rounds/list.html`

---

### 📋 ฟอร์มเพิ่ม/แก้ไขรอบรถ (Add/Edit Bus Round Form)

#### ฟิลด์ที่ 1: **เลือกทริป** (Trip Selection)
```
ชื่อ: Trip / tripId
ประเภท: Dropdown (เลือกจากรายการ Trip ที่มี)
บังคับกรอก: ใช่ (Required)
แสดงรายการ: ชื่อ Trip ที่ Active ทั้งหมด
```
**ทำหน้าที่**: เลือกว่าบรรทัดรถนี้จะไป Trip ไหน
**ตัวอย่าง**:
```
▼ เลือกทริป:
  ☐ เกาะลันตา ดำน้ำ 4 วัน 3 คืน
  ☐ ภูเก็ต 3 วัน 2 คืน
  ☑ เชียงใหม่ ผู้ชี้ชาว 3 วัน
  ☐ กระบี่จ่อมจอม 5 วัน
```
**ข้อควรระวัง**:
- ✓ เลือกให้ถูกทริป (ห้ามสลับ)
- ✓ ทริปต้อง Active ด้วย

---

#### ฟิลด์ที่ 2: **เลือกเส้นทาง (RoudeStack)** (Route Stack Selection)
```
ชื่อ: RoudeStack / roudeStackId
ประเภท: Dropdown (เลือกจากรายการ)
บังคับกรอก: ใช่ (Required)
```
**ทำหน้าที่**: เลือกเส้นทางหลายวัน (ถ้า Trip มีหลายวัน)
**อธิบาย RoudeStack**: 
- Trip เดียว อาจมี route หลายเส้น
- Roudestack = ข้อมูลเส้นทาง (เช่น "Chiang Mai Circle Tour 3 Days" อาจมี "Starting North" "Starting West" ฯลฯ)

**ตัวอย่าง**:
```
Trip: "เชียงใหม่ 3 วัน"
RoudeStack ตัวเลือก:
  ☐ วันศุกร์-อาทิตย์ (เส้นทางตอนเหนือ)
  ☑ วันอังคาร-พฤหัสบดี (เส้นทางตอนตะวันตก)
  ☐ วันจันทร์-พุธ (เส้นทางเมืองเก่า)
```

---

#### ฟิลด์ที่ 3: **หมายเลขรถ** (Bus Number)
```
ชื่อ: Bus Number
ประเภท: Number (ตัวเลข)
ค่าปกติ: 1, 2, 3, ... (หมายเลขรถ)
บังคับกรอก: ไม่บังคับ
ค่าตั้งต้น: 1
```
**ทำหน้าที่**: ระบุว่ารถคันไหน (ปกติตั้ง 1-4 คัน ต่อทริป)
**ตัวอย่าง**:
- Bus Number 1 = ชื่อรถว่า "รถที่ 1 เชียงใหม่"
- Bus Number 2 = ชื่อรถว่า "รถที่ 2 เชียงใหม่"
- Bus Number 3 = ชื่อรถว่า "รถที่ 3 เชียงใหม่"

**ข้อควรระวัง**:
- ✓ หมายเลขนี้ไม่จำเป็นต้องเป็นหมายเลขจริงของรถ
- ✓ ใช้เพื่อจำแนกรถ ไม่ใช่เพื่อข้อมูลป้ายทะเบียน

---

#### ฟิลด์ที่ 4: **จุดเริ่มต้น** (Start Point)
```
ชื่อ: Start Point
ประเภท: Text
ตัวอย่าง: "สถานีขนส่งสายตะวันออก", "โรงแรมแพนเนล", "สนามบิน"
บังคับกรอก: ไม่บังคับ
```
**ทำหน้าที่**: บอกว่ารถจะออกจากไหน (สถานี/โรงแรม)
**ตัวอย่าง**:
- "สถานีขนส่งสายตะวันออก (รั้ว)ประตู 1"
- "โรงแรม The Landmark (เสริมมพร้อม)"
- "สนามบิน สุวรรณภูมิ (ชั้น 1 ประตูทำเนียบรอพระ)"

---

#### ฟิลด์ที่ 5: **วันเดินทาง** (Departure Date)
```
ชื่อ: Departure Date
ประเภท: Date & Time (วันที่และเวลา)
รูปแบบ: YYYY-MM-DD HH:MM
บังคับกรอก: ใช่ (Required)
```
**ทำหน้าที่**: ระบุวันและเวลาที่ออกเดินทาง
**ตัวอย่าง**:
- 2025-05-16 06:00 (16 พฤษภาคม 2025 เวลา 06:00 น.)
- 2025-06-01 18:00 (1 มิถุนายน 2025 เวลา 18:00 น.)

**ข้อควรระวัง**:
- ✓ ตั้งเวลาให้สมเหตุสมผล (ปกติ 06:00-08:00 เช้า ไปทัวร์)
- ✓ หรือ 18:00-20:00 สำหรับทัวร์ค่ำคืน
- ✓ เช็คปฏิทินให้แน่ใจว่าไม่ตรงกับวันหยุด (ถ้าจำเป็น)

---

#### ฟิลด์ที่ 6: **ระยะเวลา** (Duration)
```
ชื่อ: Duration
ประเภท: Text
ตัวอย่าง: "8 ชั่วโมง", "6.5 ชั่วโมง", "4 ชั่วโมง"
บังคับกรอก: ไม่บังคับ
```
**ทำหน้าที่**: เวลาที่ใช้ในการเดินทาง (ไปถึงจุดหมาย)
**ตัวอย่าง**:
- "8 ชั่วโมง" = ออก 06:00 ถึง 14:00
- "6 ชั่วโมง" = ออก 06:00 ถึง 12:00
- "10 ชั่วโมง" = ออก 08:00 ถึง 18:00

**ข้อควรระวัง**:
- ✓ คำนวณให้รวมเวลาหยุดพัก/กิน
- ✓ บวกเวลา Buffer 30 นาที (ปัญหาเบื่อจราจร ฯลฯ)
- ✓ ห้ามตั้งน้อยเกินไป (ลูกค้ากลัว)

---

#### ฟิลด์ที่ 7: **จำนวนที่นั่งทั้งหมด** (Total Seats)
```
ชื่อ: Total Seats
ประเภท: Number (ตัวเลข)
ช่วงปกติ: 30-50 (รถขนาดปกติ)
บังคับกรอก: ใช่ (Required)
```
**ทำหน้าที่**: บอกว่ารถคันนี้มีที่นั่งกี่ที่
**ความสำคัญ**: 
- 🔴 ตัวเลขนี้คือ **สูงสุด** ที่ลูกค้าจองได้
- หากตั้ง 50 ที่ → ลูกค้าจองได้ได้ 50 คน (ไม่เกิน)

**ตัวอย่าง**:
- รถขนาด VIP: Total Seats = 30 (นั่งสบาย)
- รถขนาด ปกติ: Total Seats = 40
- รถขนาด เล็ก: Total Seats = 20

**ข้อควรระวัง**:
- ✓ ตั้งให้สอดคล้องกับรถจริง
- ✓ ไม่ตั้งเกินกำลังรถ (เสี่ยงปัญหา/อนุรักษ์/พิษ)
- ❌ ห้ามตั้งให้น้อยเกินไป (เสียโอกาส)

---

#### ฟิลด์ที่ 8: **ที่นั่งที่จองแล้ว** (Booked Seats)
```
ชื่อ: Booked Seats
ประเภท: Number (ตัวเลข อ่านได้อย่างเดียว)
ค่าตั้งต้น: 0
อัปเดต: อัตโนมัติ (ไม่ต้องใส่ด้วยตัวเอง)
```
**ทำหน้าที่**: นับจำนวนที่นั่งที่ลูกค้าจองไปแล้ว (อัตโนมัติ)
**ตัวอย่าง**:
- Booked Seats = 15 → เหลือ 35 ที่นั่ง
- Booked Seats = 40 → เหลือ 10 ที่นั่ง
- Booked Seats = 50 → FULL (เต็ม) ปิดจองแล้ว

**ข้อควรระวัง**:
- ✓ ระบบอัปเดตอัตโนมัติ ไม่ต้องแก้มือ
- ✓ ดูตรวจสอบความเป็นไปได้ (ว่าเพิ่มขึ้นถูกต้อง)

---

#### ฟิลด์ที่ 9: **สถานะการรับจอง** (Is Open)
```
ชื่อ: Is Open
ประเภท: Checkbox (ติ๊ก/ไม่ติ๊ก)
ตัวเลือก:
  ☑️ Open (รับจองอยู่)
  ☐ Closed (ปิดจอง)
บังคับกรอก: ไม่บังคับ
ค่าตั้งต้น: Open
```
**ทำหน้าที่**: 
- ✅ Open = ลูกค้าสามารถจองทริปนี้ได้
- ❌ Closed = ปิดการจอง (ทั้งที่อาจยังมีที่นั่งว่าง)

**เหตุผลการปิด**:
- 🔴 รถเต็ม (Booked Seats = Total Seats)
- ⚠️ ยกเลิกหรือเลื่อนทริป
- ⚠️ เหตุวิสัย (ซ่อมรถ, เปลี่ยนตารางการ)

**ข้อควรระวัง**:
- ✓ ตั้งเป็น "Open" โดยค่าตั้งต้น เมื่อสร้าง
- ✓ เปลี่ยนเป็น "Closed" เมื่อเต็มหรือมีปัญหา
- ❌ ห้าวจำเป็น ห้ามเปลี่ยนซ้ำๆ (ลูกค้างง)

---

#### ฟิลด์ที่ 10: **ผู้รับผิดชอบ** (Responsible Person)
```
ชื่อ: Responsible Person
ประเภท: Text (ชื่อ/หมายเลขประจำตัว)
ตัวอย่าง: "สมชาย ลิ้มสกุล", "ชาตรี", "เบอร์ 09999xxxxx"
บังคับกรอก: ไม่บังคับ
```
**ทำหน้าที่**: ผู้นำทัวร์ หรือ ผู้รับผิดชอบคนนี้
**ตัวอย่าง**:
- "สมชาย ลิ้มสกุล (ผู้นำทัวร์)" 
- "เบอร์โทร 081-234-5678 (ก่องประชาสัมพันธ์)"
- "ไกรพัฒน์ (ไกด์เชียงใหม่)"

**ข้อควรระวัง**:
- ✓ เขียนชื่อหรือหมายเลขที่ติดต่อได้
- ✓ เพื่อให้ลูกค้าติดต่อได้หากเกิดปัญหา

---

#### ฟิลด์ที่ 11: **ราคาเพิ่มเติม** (Extra Price)
```
ชื่อ: Extra Price
ประเภท: Number (ตัวเลข)
หน่วย: บาท
บังคับกรอก: ไม่บังคับ
ค่าตั้งต้น: 0
```
**ทำหน้าที่**: เพิ่มราคาตามรอบ (บวกกับราคา Trip ปกติ)
**เหตุผล**:
- 💰 ราคาที่พัก แพง (ฤดูท่องเที่ยว)
- 💰 ต้นทุนเชื้อเพลิงเพิ่มขึ้น
- 💰 การให้บริการพิเศษ (ก่องปลาดำน้ำ ฯลฯ)

**ตัวอย่าง**:
```
Trip ปกติราคา: 3,500 บาท/คน
Bus Round วันศุกร์ (ฤดูท่องเที่ยว):
  Extra Price: 500 บาท
  ราคาจริง: 3,500 + 500 = 4,000 บาท/คน

Bus Round วันอังคาร (นอกฤดู):
  Extra Price: 0 บาท (ไม่เพิ่ม)
  ราคาจริง: 3,500 บาท/คน
```

**ข้อควรระวัง**:
- ✓ ตั้งให้สมเหตุสมผล
- ✓ ไม่ตั้งสูงเกินไป (ลูกค้ายิ่งไม่ซื้อ)
- ✓ หากไม่เพิ่มราคา ตั้งเป็น 0 หรือว่าง

#### ฟิลด์ที่ 12: **จุดรับส่ง** (Pickup Points)
```
ชื่อ: Pickup Points
ประเภท: JSON (ข้อมูลแบบเก็บไว้)
รูปแบบ: Array ของ locations
ตัวอย่าง:

  {"location": "สถานีตะวันออก", "time": "06:00", "price": 0},

**ทำหน้าที่**: ระบุจุดรับ-ส่งผู้โดยสาร และอาจเรียกเพิ่มเติม
**ตัวอย่าง**:
- จุดที่ 1: "สถานีตะวันออก" (06:00) ฟรี
- จุดที่ 2: "โรงแรมเอ" (06:30) เรียกเพิ่ม 50 บาท
- จุดที่ 3: "สนามบิน" (07:00) เรียกเพิ่ม 100 บาท

**ความสำคัญ**:
- 📍 ลูกค้ากำหนดจุดรับที่สะดวก
- 💰 บางจุดอาจเรียกค่าพิเศษ (ห่างไกล)

**ข้อควรระวัง**:
- ✓ บอกชัดเจนว่าจุดใดต้องรอลูกค้า กี่นาที
- ✓ เวลารับต้องสมเหตุสมผล (15-30 นาทีระหว่างจุด)
- ✓ ระบุตำแหน่งให้ชัดเจน (ใช้สถานีขนส่ง/โรงแรมปลอดภัย)

#### ฟิลด์ที่ 13: **สถานะ** (Is Active/Published)
```
ชื่อ: Status / Is Active

ตัวเลือก:
  ☑️ Published / Active (ลูกค้าเห็นและจองได้)
