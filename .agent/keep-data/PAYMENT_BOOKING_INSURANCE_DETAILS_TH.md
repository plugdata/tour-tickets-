## 📍 ส่วนที่ 1: BOOKING MANAGEMENT (การจัดการการจอง)

### หน้า: `bookings/list.html` และ `booking/form.html`

---

### 📋 ฟอร์มสร้างการจองใหม่ (Create Booking Form)

#### ฟิลด์ที่ 1: **เลือกลูกค้า** (Customer Selection)
```
ชื่อ: Customer / userId
ประเภท: Dropdown หรือ Search
บังคับกรอก: ใช่ (Required)
แสดง: "ชื่อ (เบอร์โทร)" เช่น "สมชาย (081-234-5678)"
```
**ทำหน้าที่**: เลือกว่าการจองนี้เป็นของลูกค้าคนไหน
**ตัวเลือก**:
- 👤 ลูกค้าเก่า (คนเคย จองมาก่อน) → ค้นหาจากชื่อ
- ➕ **ลูกค้าใหม่** → สร้างลูกค้าใหม่ก่อน แล้วค่อยจอง

**ฟิลด์ที่ต้องเตรียม (สำหรับลูกค้าใหม่)**:
```
├─ ชื่อเต็ม (Name)
├─ หมายเลขโทรศัพท์ (Phone)
├─ อีเมล (Email)
├─ Username (สำหรับ login)
└─ Password (สำหรับ login)
```

---

#### ฟิลด์ที่ 2: **เลือกทริป** (Trip Selection)
```
ชื่อ: Trip / tripId (ผ่าน BusRound)
ประเภท: Dropdown
บังคับกรอก: ใช่ (Required)
แสดง: ชื่อ Trip ที่ Active ทั้งหมด
```
**ทำหน้าที่**: เลือกทริปที่ลูกค้าอยากจอง
**ตัวอย่าง**:
```
▼ เลือกทริป:
  ☐ เกาะลันตา ดำน้ำ 4 วัน 3 คืน
  ☑ เชียงใหม่ ผู้ชี้ชาว 3 วัน
  ☐ ภูเก็ต 3 วัน 2 คืน
```

---

#### ฟิลด์ที่ 3: **เลือกรอบรถ** (Bus Round Selection)
```
ชื่อ: Bus Round / busRoundId
ประเภท: Dropdown
บังคับกรอก: ใช่ (Required)
แสดง: วันเดินทาง + ราคา + ที่นั่งเหลือ
ตัวอย่าง:
  ☐ 2025-05-16 (ศุกร์) 4,000บ - 35/40 ที่
  ☑ 2025-05-19 (จันทร์) 3,500บ - 46/50 ที่
```
**ทำหน้าที่**: เลือกวันเดินทางและรอบรถ (กำหนดราคาและจำนวนที่นั่ง)

---

#### ฟิลด์ที่ 4: **จำนวนที่นั่ง** (Number of Seats)
```
ชื่อ: Seats
ประเภท: Number (ตัวเลข)
ช่วง: 1-50
บังคับกรอก: ใช่ (Required)
ค่าตั้งต้น: 1
```
**ทำหน้าที่**: ลูกค้าจองกี่ที่นั่ง
**ตัวอย่าง**:
- "2 ที่" → ลูกค้า 2 คน
- "4 ที่" → ครอบครัว 4 คน
- "1 ที่" → คนเดียว

**เมื่อลูกค้าใส่จำนวน**:
```
ระบบคำนวณอัตโนมัติ:
Seats = 2
BusRound Price = 3,500 บาท
Total Amount = 2 × 3,500 = 7,000 บาท
```

---

#### ฟิลด์ที่ 5: **ประเภทการจอง** (Booking Type)
```
ชื่อ: Booking Type
ประเภท: Dropdown
ตัวเลือก:
  ☐ SINGLE (จองเดี่ยว)
  ☑ GROUP (จองกลุ่ม)
  ☐ CORPORATE (บริษัท/หน่วยงาน)
บังคับกรอก: ไม่บังคับ
ค่าตั้งต้น: SINGLE
```
**ทำหน้าที่**: ประเภทของลูกค้า (เพื่อวัตถุประสงค์ในการบริหาร)
**การใช้งาน**:
- **SINGLE**: คนเดียว ครอบครัวเล็ก (1-4 คน)
- **GROUP**: กลุ่มใหญ่ (5+ คน) อาจมีราคาลด
- **CORPORATE**: บริษัท อาจมีเงื่อนไขพิเศษ

**ประโยชน์**:
- 📊 วิเคราะห์ลูกค้า
- 💰 ใช้กรองสำหรับโปรโมชัน
- 📋 รายงานตามประเภท

---

#### ฟิลด์ที่ 6: **อาหารที่แพ้** (Food Allergy)
```
ชื่อ: Food Allergy
ประเภท: Text Area (ข้อความยาว)
ตัวอย่าง: "แพ้ปลา แพ้กุ้ง", "เวจ", "ไม่กินปลา"
บังคับกรอก: ไม่บังคับ (Optional)
```
**ทำหน้าที่**: บอกสิ่งที่ลูกค้าแพ้/ไม่ทาน เพื่อจัดอาหาร
**ตัวอย่าง**:
```
"แพ้นาง แพ้กุ้ง แพ้กรรม"
"เวจ - ไม่กินเนื้อสัตว์ทั้งหมด"
"แพ้รีด/อาหารติดน้ำหนึ่ง"
"ไม่ชอบ: ผัก บวก พืชผักแบบเอกสิทธิ"
```

**ข้อควรระวัง**:
- ✓ เขียนชัดเจนให้เข้าใจ
- ✓ โทรยืนยันอีกครั้งก่อนเดินทาง
- ✓ ส่งไปให้เพื่อนรับหลักสินค้า เพื่อเตรียมอาหาร

---

#### ฟิลด์ที่ 7: **จำนวนเงินรวม** (Total Amount)
```
ชื่อ: Total Amount
ประเภท: Number (อ่านได้อย่างเดียว)
คำนวณ: Seats × Bus Round Price
```
**ทำหน้าที่**: ราคารวมทั้งหมดที่ลูกค้าต้องจ่าย
**ตัวอย่าง**:
```
Seats: 3 คน
Bus Round Price: 3,500 บาท/คน
Total Amount: 3 × 3,500 = 10,500 บาท
```

**ข้อควรระวัง**:
- ✓ ระบบคำนวณอัตโนมัติ ไม่ต้องใส่ด้วยตัวเอง
- ✓ เช็คให้แน่ใจว่าถูกต้อง

---

#### ฟิลด์ที่ 8: **ผู้บันทึก** (Recorded By)
```
ชื่อ: Recorded By
ประเภท: Text (ชื่อเจ้าหน้าที่)
บังคับกรอก: ไม่บังคับ
ค่าตั้งต้น: ชื่อสมาชิกที่ login อยู่
```
**ทำหน้าที่**: บันทึกว่าเจ้าหน้าที่คนไหนสร้างการจองนี้
**ตัวอย่าง**: "สมชาย ลิ้มสกุล" (สำนักพิมพ์ 081-234-5678)
**ประโยชน์**:
- 📊 รู้ว่าใครรับโทรศัพท์/จดรายการ
- 💰 คำนวณคอมมิชชั่น
- 📋 ติดตามผู้รับผิดชอบ

---

#### ฟิลด์ที่ 9: **สถานะการจอง** (Booking Status)
```
ชื่อ: Status
ประเภท: Dropdown (เปลี่ยนได้เมื่อต้องการ)
ตัวเลือก:
  ☑ PENDING (รอการชำระเงิน/อนุมัติ)
  ☐ CONFIRMED (อนุมัติแล้ว เตรียมเดินทาง)
  ☐ CANCELLED (ยกเลิก)
บังคับกรอก: ใช่
ค่าตั้งต้น: PENDING
```
**ทำหน้าที่**: สถานะของการจองที่แต่ละขั้นตอน
**การไหลของสถานะ**:
```
สร้างการจอง
    ↓
PENDING (รอชำระเงิน)
    ↓
ลูกค้าอัพโหลดสลิป
    ↓
เจ้าหน้าที่ยืนยันการชำระ
    ↓
CONFIRMED (ยืนยันแล้ว, เตรียมเดินทาง)
    ↓
ไป tour
    ↓
COMPLETED (สำเร็จ) [ไม่ใช่เลือกในที่นี้]

หรือ

PENDING → CANCELLED (ลูกค้าขอยกเลิก/ปัญหา)
```

**ข้อควรระวัง**:
- ✓ ตั้งเป็น PENDING โดยค่าตั้งต้น เมื่อสร้างการจอง
- ✓ เปลี่ยนเป็น CONFIRMED เมื่อชำระเงินและอนุมัติแล้ว
- ✓ เปลี่ยนเป็น CANCELLED ถ้าลูกค้าขอยกเลิก

## 📍 ส่วนที่ 2: PAYMENT MANAGEMENT (การจัดการการชำระเงิน)

### หน้า: `payments/list.html`

---

### 💳 ขั้นตอนการชำระเงินและการยืนยัน

#### ขั้นตอนที่ 1: **ลูกค้าอัพโหลดสลิปการโอนเงิน**

```
บนเว็บไซต์ลูกค้า:
1. ไปที่หน้า "ตรวจสอบ/อัพเดทสลิป"
2. อัพโหลดรูปสลิปการโอนเงิน
3. ระบบบันทึก:
   ├─ slipUrl (เก็บรูปสลิป)
   ├─ Payment.status = "PENDING" (รอการยืนยัน)
   └─ createdAt = วันที่อัพโหลด
```

---

#### ขั้นตอนที่ 2: **เจ้าหน้าที่ตรวจสอบสลิป**

**หน้า**: `payments/list.html`

**ฟิลด์ที่ต้องเช็ค**:

```
ชิ้น No. 1: ปริมาณการโอนเงิน
  ├─ ต้อง = Booking.totalAmount
  ├─ ตัวอย่าง: ต้อง 7,000 บาท, ลูกค้าโอน 7,000 บาท ✓
  └─ ถ้าต่างกัน: โอน 6,500 บาท ✗ (ขาด 500 บาท)

ชิ้น No. 2: บัญชีธนาคารปลายทาง
  ├─ ต้อง = หมายเลขบัญชีของบริษัท (ในระบบ)
  ├─ ตัวอย่าง: ต้อง "ธนาคารกรุงเทพ 234-5678-9", โอนมา "ธนาคารกสิกรไทย 100-1234-5" ✗
  └─ ต้อง = Bank Account ที่ Active ในระบบ

ชิ้น No. 3: วันที่โอนเงิน
  ├─ ต้อง = วันเดินทางจะต้องเป็นวันหลังนี้ (อย่างน้อย 2-3 วัน)
  ├─ ตัวอย่าง: วันเดินทาง 2025-05-19, โอนเงิน 2025-05-16 ✓
  └─ ถ้าโอนนาน: โอน 2025-04-20 (1 เดือนก่อน) ยังโอเค

ชิ้น No. 4: เลขที่ธุรกรรม/ลำดับเลข
  ├─ บอก: สลิปต้องมีเลขอ้างอิง (reference number)
  ├─ ตัวอย่าง: "Ref: 2025051612345678"
  └─ บอกให้ลูกค้าจดไว้ เพื่ออ้างอิง
```

---

#### ขั้นตอนที่ 3: **อนุมัติหรือปฏิเสธ**

**ถ้าเช็ครายละเอียดถูกต้องทั้งหมด**:
```
คลิก "APPROVE"
↓
ระบบอัปเดต:
├─ Payment.status = "VERIFIED" (ยืนยันแล้ว)
├─ Payment.verifiedBy = "ชื่อเจ้าหน้าที่"
├─ Payment.verifiedDate = วันที่ยืนยัน
└─ เจ้าหน้าที่สามารถอนุมัติ Booking ได้แล้ว
```

**ถ้าพบปัญหา**:
```
คลิก "REJECT"
↓
ระบบอัปเดต:
├─ Payment.status = "REJECTED" (ไม่ผ่าน)
├─ เจ้าหน้าที่พิมพ์เหตุผลที่ปฏิเสธ
├─ ส่งอีเมลไปลูกค้า
└─ ลูกค้าต้องอัพโหลดสลิปใหม่
```

---

### 📋 ฟิลด์ของ Payment Record

```
ชื่อ: Payment
─────────────────────────────────────

paymentId (ID)
  ├─ รหัสอ้างอิง (1, 2, 3, ...)
  └─ สร้างอัตโนมัติ

bookingId (เชื่อมกับ Booking)
  ├─ ระบุว่า Payment นี้เป็นของ Booking ไหน
  └─ ตัวอย่าง: Booking No. 42

userId (เชื่อมกับ User)
  ├─ ระบุว่า Payment นี้เป็นของลูกค้าคนไหน
  └─ ตัวอย่าง: ลูกค้า No. 15 (สมชาย สมการ)

transactionId (รหัสการโอนเงิน)
  ├─ reference number จากสลิป
  ├─ ตัวอย่าง: "BKK2025051612345"
  └─ เก็บเพื่ออ้างอิง

amount (จำนวนเงิน)
  ├─ เงินที่ลูกค้าโอนมา
  ├─ ตัวอย่าง: 7,000 บาท
  └─ ต้องตรงกับ Booking.totalAmount

slipUrl (รูปสลิป)
  ├─ URL ของรูปสลิปการโอน
  ├─ สำหรับเจ้าหน้าที่ตรวจสอบ
  └─ เก็บไว้ 1 ปีเพื่อ audit

status (สถานะการชำระ)
  ├─ PENDING: รอการตรวจสอบ
  ├─ VERIFIED: ผ่านการตรวจสอบ (ยืนยันแล้ว)
  └─ REJECTED: ไม่ผ่าน ต้องโอนใหม่

verifiedBy (เจ้าหน้าที่ยืนยัน)
  ├─ ชื่อของเจ้าหน้าที่ที่อนุมัติ
  ├─ ตัวอย่าง: "ธนา สมแข็ง"
  └─ บันทึกอัตโนมัติ

verifiedDate (วันที่ยืนยัน)
  ├─ วันที่เจ้าหน้าที่อนุมัติ
  ├─ ตัวอย่าง: "2025-05-17 14:30"
  └─ บันทึกอัตโนมัติ

createdAt (วันที่อัพโหลด)
  ├─ วันที่ลูกค้าอัพโหลดสลิป
  ├─ ตัวอย่าง: "2025-05-16 15:45"
  └─ บันทึกอัตโนมัติ
```

```
ประกัน Type 1: "ประกันการบินนานาชาติ"

└─ ครอบคลุม: 100% มัดจำ
```

### 📋 ฟอร์มเลือกประกัน (Customer Insurance Form)

**ขั้นตอนที่ 1: ลูกค้ากรอกข้อมูลส่วนตัว**

```
ชื่อ-สกุล: "สมชาย สมการ"
อายุ: 35
เลขบัตรประชาชน: "1-2345-67890-12-3"
เพศ: "ชาย"
กลุ่มเลือด: "O"
วันเกิด: "1990-05-16"
```

**ข้อควรระวัง**:
- ✓ ต้องตรงกับบัตรประชาชน
- ✓ ประกันมีเงื่อนไขตามอายุ (เช่น อายุ >70 อาจไม่คุ้มบางประเภท)

---

**ขั้นตอนที่ 2: เลือกประเภทประกัน**

### 🎯 สรุปขั้นตอนการจองจบสิ้น

```
1. สร้างการจอง (Booking Form)
   ├─ เลือกลูกค้า / Trip / Bus Round
   ├─ ระบุจำนวนที่นั่ง
   ├─ เขียนอาหารที่แพ้
   └─ Status: PENDING

2. ลูกค้าสมบูรณ์รายละเอียด (บนเว็บ)
   ├─ อัพโหลดสลิปการโอนเงิน
   ├─ กรอกฟอร์มประกันภัย
   └─ ส่งข้อมูลให้

3. เจ้าหน้าที่ตรวจสอบ (Payment Form)
   ├─ เช็คจำนวนเงิน
   ├─ เช็คบัญชีธนาคาร
   ├─ เช็ควันที่โอน
   └─ อนุมัติ → Payment.status = VERIFIED

4. อนุมัติการจอง (Booking Status)
   ├─ เช็คทั้งหมด OK
   ├─ เปลี่ยน Status: CONFIRMED
   └─ ส่งอีเมลยืนยันไปลูกค้า

5. เตรียมเดินทาง
   ├─ 1 สัปดาห์ก่อน: โทรลูกค้าเตือน
   ├─ 1 วันก่อน: ส่งเอกสารประจำตัว
   ├─ วันเดินทาง: มาเก็บตัวเวลาที่กำหนด
   └─ ไปทัวร์ได้!
```
