## 📍 ส่วนที่ 5: CORE WORKFLOW FOR BACKOFFICE

### 📊 ขั้นตอนการจองแบบเต็มสำหรับแบคออฟฟิส

```
┌────────────────────────────────────────────────────────────────┐
│                    BACKOFFICE BOOKING WORKFLOW                │
└────────────────────────────────────────────────────────────────┘

ขั้นตอนที่ 1: ลูกค้าโทรเข้ามา
═════════════════════════════════════════════════════════════════
Staff → Dashboard (ดูว่าเว็บไซต์ว่างไปไหนบ้าง)
     ↓
Staff → Trips/List (ค้นหาทริปที่ลูกค้าสนใจ)
     ↓
Staff → Bus-Rounds/List (เช็ควันเดินทาง, ที่นั่ง, ราคา)
     ↓
Staff → ตอบลูกค้า ("ทริป ... ราคา ... วันไหนได้บ้าง")
     └─> Reference: TRIP_BUS_FORM_DETAILS_TH.md

ขั้นตอนที่ 2: ลูกค้าตัดสินใจจอง
═════════════════════════════════════════════════════════════════
Staff → Bookings/Form (สร้างการจองใหม่)
     ├─ เลือกลูกค้า (หรือสร้างลูกค้าใหม่)
     ├─ เลือก Trip
     ├─ เลือก Bus Round
     ├─ ใส่จำนวนที่นั่ง
     ├─ เขียนอาหารที่แพ้
     └─ Status: PENDING
     
Staff → บอกลูกค้าราคาและมัดจำ
     ├─ "ราคารวม: ..." บาท
     ├─ "มัดจำ: ..." บาท"
     └─ "โอนไปบัญชี: ..."
     └─> Reference: PAYMENT_BOOKING_INSURANCE_DETAILS_TH.md

ขั้นตอนที่ 3: ลูกค้าส่งหลักฐานการชำระเงิน
═════════════════════════════════════════════════════════════════
ลูกค้า (Website) → Upload Payment Slip
     ↓
Payment Record สร้าง (status = PENDING)
     ↓
Staff → Payments/List (ตรวจสอบสลิป)
     ├─ ดูรูปสลิป
     ├─ เช็ค: จำนวน, บัญชี, วันที่
     └─> Reference: PAYMENT_BOOKING_INSURANCE_DETAILS_TH.md
     
Staff → APPROVE Payment
     └─ Payment.status = VERIFIED
        Payment.verifiedBy = Staff name
        Payment.verifiedDate = now()

ขั้นตอนที่ 4: ลูกค้ากรอกประกันภัย
═════════════════════════════════════════════════════════════════
ลูกค้า (Website) → Insurance Form
     ├─ ใส่ข้อมูลส่วนตัว
     ├─ เลือกประเภทประกัน
     └─ ยอมรับเงื่อนไข
     
InsuranceForm Record สร้าง
     ↓
Staff → Insurance/List (ดูว่ากรอกหรือยัง)

ขั้นตอนที่ 5: Staff อนุมัติการจอง
═════════════════════════════════════════════════════════════════
Staff → Bookings/View (ตรวจสอบข้อมูลครบถ้วน)
     ├─ ✓ Payment VERIFIED แล้ว
     ├─ ✓ Insurance Form กรอกแล้ว
     ├─ ✓ ที่นั่งยังว่างอยู่
     └─ ✓ ทั้งหมด OK
     
Staff → Approve Booking
     ├─ Booking.status = CONFIRMED
     ├─ SeatBooking records สร้าง (assign seats)
     └─ BusRound.bookedSeats += number of seats
     
Staff → Send Email/SMS
     ├─ Subject: "ยืนยันการจองทัวร์"
     └─ Body: "Booking #..., ราคา ..., วันเดินทาง ..."

ขั้นตอนที่ 6: ติดตามก่อนเดินทาง
═════════════════════════════════════════════════════════════════
7 วันก่อน → Staff โทรเตือนลูกค้า
3 วันก่อน → ส่ง Itinerary + เอกสาร
1 วันก่อน → เตียมรถ/เจ้าหน้าที่
วันเดินทาง → ตรวจสอบลูกค้ามา กรุณาเก็บตัว

ขั้นตอนที่ 7: เดินทาง
═════════════════════════════════════════════════════════════════
ลูกค้า Booking.status = COMPLETED
Staff → Reports (เพิ่มรายได้ + Commission)

ขั้นตอนที่ 8: หลังจากกลับมา
═════════════════════════════════════════════════════════════════
Staff → ส่งแบบประเมินความพึงพอใจ
ลูกค้า → ตอบแบบ
Staff → Reviews/Feedback (บันทึก)
```

### 🔄 Decision Tree สำหรับ Staff

```
ลูกค้าโทรมาจอง
     ↓
[ลูกค้ากรรม?]
     ├─ NO: สร้างลูกค้าใหม่ (User form)
     └─ YES: ตรวจสอบข้อมูล

[ทริปไหน?]
     ├─ Trip ว่าง? YES → ไปต่อ
     └─ Trip เต็ม? NO → แนะนำทริปอื่น

[วันไหน?]
     ├─ Bus Round ว่าง? YES → ไปต่อ
     └─ เต็มแล้ว? NO → แนะนำวันอื่น

[คนกี่คน?]
     ├─ ที่นั่งพอ? YES → ไปต่อ
     └─ ไม่พอ? NO → ลดจำนวนหรือ bus รอบอื่น

[เลือก OK?]
     ├─ YES: บันทึก Booking (PENDING)
     └─ NO: ปิดโทรศัพท์ ลองอีกที

[โอนเงินแล้ว?]
     ├─ YES: ตรวจสอบ Payment → APPROVE
     └─ NO: ส่ง Invoice + Bank Account

[ประกันหรือไม่?]
     ├─ YES: ลูกค้ากรอก Insurance Form
     └─ NO: ข้ามขั้นตอนนี้

[ทั้งหมด OK?]
     ├─ YES: CONFIRM Booking → ส่ง Email
     └─ NO: รอข้อมูล + ติดตาม
```
